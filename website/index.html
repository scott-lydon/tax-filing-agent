The four pillars

1. Chat loop with state

Sessions and messages persist to SQLite; an in-flight session rehydrates after a restart so the agent can still quote turn one.

2. Real tools

record_w2, record_answer, compute_return, finalize_and_render, ask_question. The last tool fills the real IRS PDF and writes it to disk.

3. Guardrails in code

Zod validation, a refusal screen, and a distinct-question counter that refuses a sixth question. Not prompt-only.

4. Observation

Every step is appended to a JSONL trail, served at a trace endpoint, and streamed into a "Show what the agent did" drawer.

Trade-offs

SQLite, not a managed DB

Accept: single instance only. Bites: multi-instance scale. Revisit: more than one web instance.

Child Tax Credit on line 19 only

Accept: refundable ACTC (line 28) not modeled. Bites: low-tax filer with kids. Revisit: add Schedule 8812. Documented in code.

Paste-in W-2 default

Accept: no OCR. Bites: reviewer who insists on image upload. Revisit: stretch goal.

Cost shape

Per completed return: a handful of Sonnet 4.6 turns (W-2 ack, five questions, finalize) dominate; hosting is the free tier; the PDF fill and tax math are local CPU.
